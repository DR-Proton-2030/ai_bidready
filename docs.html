Overview

The BidReady AI Model API provides powerful floor plan object detection capabilities using state-of-the-art YOLOv8 technology. Upload an image of a floor plan and get detailed analysis of architectural elements including doors, windows, walls, and more.

Key Features

  • 9 Object Types: Detect columns, curtain walls, dimensions, doors, railings, sliding doors, staircases, walls, and windows
  • Confidence Scoring: Adjustable detection confidence thresholds (0.0 - 1.0)
  • Selective Detection: Choose which object types to detect
  • Annotated Images: Get back annotated images with bounding boxes
  • Object Counting: Automatic counting of detected objects
  • RESTful API: Easy integration with any web application
Base URLs:
• Local Development: http://localhost:8000
• Network Access: http://192.168.1.38:8000

Quick Start

1. Health Check

First, verify the API is running:

curl http://localhost:8000/health

2. Simple Detection

Upload an image for detection with default settings:

curl -X POST "http://localhost:8000/detect-simple" \
     -H "Content-Type: multipart/form-data" \
     -F "file=@your_floorplan.jpg"

3. Advanced Detection

Customize confidence and select specific objects:

curl -X POST "http://localhost:8000/detect?confidence=0.7&selected_labels=Door,Window" \
     -H "Content-Type: multipart/form-data" \
     -F "file=@your_floorplan.jpg"

API Endpoints

GET /

Returns API information and available endpoints.

GET /health

Health check endpoint. Returns API status and model loading status.

GET /labels

Get list of all available detection labels.

POST /detect

Full object detection with customizable parameters.

Parameters:

Parameter Type Required Description
file File Required Image file (JPG, JPEG, PNG)
confidence Float Optional Detection threshold (0.0-1.0, default: 0.25)
selected_labels String Optional Comma-separated list of labels to detect
POST /detect-simple

Simplified detection endpoint using default parameters (confidence=0.25, all labels).

Parameters:

Parameter Type Required Description
file File Required Image file (JPG, JPEG, PNG)

Response Format

All detection endpoints return JSON in the following format:

{
  "success": true,
  "total_detections": 5,
  "object_counts": {
    "Door": 2,
    "Window": 3
  },
  "detections": [
    {
      "label": "Door",
      "confidence": 0.85,
      "bbox": {
        "x1": 100.5,
        "y1": 200.3,
        "x2": 150.8,
        "y2": 280.1
      }
    }
  ],
  "annotated_image": "data:image/png;base64,iVBORw0KGgo...",
  "parameters": {
    "confidence": 0.25,
    "selected_labels": ["Door", "Window", "Wall"]
  }
}

Usage Examples

JavaScript/Fetch API

// Simple detection
const formData = new FormData();
formData.append('file', imageFile);

fetch('http://localhost:8000/detect-simple', {
    method: 'POST',
    body: formData
})
.then(response => response.json())
.then(data => {
    console.log('Object counts:', data.object_counts);
    console.log('Total detections:', data.total_detections);
    
    // Display annotated image
    document.getElementById('result-img').src = data.annotated_image;
})
.catch(error => console.error('Error:', error));

Python Requests

import requests

# Upload and detect
with open('floorplan.jpg', 'rb') as file:
    response = requests.post(
        'http://localhost:8000/detect',
        files={'file': file},
        params={
            'confidence': 0.5,
            'selected_labels': 'Door,Window,Wall'
        }
    )

result = response.json()
print(f"Found {result['total_detections']} objects:")
for label, count in result['object_counts'].items():
    print(f"- {label}: {count}")

jQuery Example

$('#upload-form').on('submit', function(e) {

                '
Results:
' +
                '
Total detections: ' + data.total_detections + '
' +
                ''
            );
        },
        error: function(xhr, status, error) {
            console.error('Detection failed:', error);
        }
    });
});

Website Integration Guide

Frontend Integration

Here's a complete example of integrating the API into your website:

<!DOCTYPE html>
<html>
<head>
    <title>Floor Plan Analyzer</title>
</head>
<body>
    <h1>Upload Floor Plan</h1>
    
    <form id="upload-form">
        <input type="file" id="image-input" accept="image/*" required>
        <label>
            Confidence: 
            <input type="range" id="confidence" min="0" max="1" step="0.1" value="0.25">
            <span id="confidence-value">0.25</span>
        </label>
        <button type="submit">Analyze Floor Plan</button>
    </form>
    
    <div id="results"></div>
    <div id="loading" style="display:none;">Processing...</div>

    <script>
        document.getElementById('upload-form').addEventListener('submit', async function(e) {
            e.preventDefault();
            
            const fileInput = document.getElementById('image-input');
            const confidence = document.getElementById('confidence').value;
            const resultsDiv = document.getElementById('results');
            const loadingDiv = document.getElementById('loading');
            
            if (!fileInput.files[0]) {
                alert('Please select an image');
                return;
            }
            
            // Show loading
            loadingDiv.style.display = 'block';
            resultsDiv.innerHTML = '';
            
            // Prepare form data
            const formData = new FormData();
            formData.append('file', fileInput.files[0]);
            
            try {
                const response = await fetch(`http://localhost:8000/detect?confidence=${confidence}`, {
                    method: 'POST',
                    body: formData
                });
                
                const data = await response.json();
                
                if (data.success) {
                    // Display results
                    let html = `<h3>Detection Results</h3>`;
                    html += `<p>Total objects found: ${data.total_detections}</p>`;
                    
                    if (Object.keys(data.object_counts).length > 0) {
                        html += `<h4>Object Counts:</h4><ul>`;
                        for (const [label, count] of Object.entries(data.object_counts)) {
                            html += `<li>${label}: ${count}</li>`;
                        }
                        html += `</ul>`;
                    }
                    
                    html += `<h4>Annotated Image:</h4>`;
                    html += `<img src="${data.annotated_image}" alt="Annotated floor plan" style="max-width: 100%;">`;
                    
                    resultsDiv.innerHTML = html;
                } else {
                    resultsDiv.innerHTML = '<p style="color: red;">Detection failed</p>';
                }
            } catch (error) {
                resultsDiv.innerHTML = `<p style="color: red;">Error: ${error.message}</p>`;
            } finally {
                loadingDiv.style.display = 'none';
            }
        });
        
        // Update confidence display
        document.getElementById('confidence').addEventListener('input', function() {
            document.getElementById('confidence-value').textContent = this.value;
        });
    </script>
</body>
</html>

CORS Configuration

Important: For production use, update the CORS settings in api.py to only allow your website's domain instead of allowing all origins (*). 
# In api.py, replace:
allow_origins=["*"]

# With your domain:
allow_origins=["https://yourdomain.com", "https://www.yourdomain.com"]

Error Handling

Common Error Codes

Status Code Error Type Description
400 Bad Request Invalid file type, confidence out of range, or invalid labels
500 Internal Server Error Model loading failed or processing error

Error Response Format

{
    "detail": "Invalid file type. Only JPG, JPEG, and PNG are supported."
}
File Size Limits: Keep uploaded images under 10MB for optimal performance. Very large images may cause timeout errors.

Performance & Limitations

Performance Tips

  • Image Size: Resize images to 1024x1024 pixels or smaller for faster processing
  • File Format: JPEG files are typically smaller and faster to process than PNG
  • Confidence Threshold: Higher confidence values (0.5-0.8) reduce false positives
  • Selective Detection: Specify only needed labels to improve speed

Current Limitations

  • Single image processing only (no batch uploads)
  • Maximum file size: 10MB
  • Supported formats: JPG, JPEG, PNG only
  • Processing time: 2-10 seconds depending on image size and complexity
